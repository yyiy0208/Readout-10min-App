## 修改段落样式与原型一致

### 问题分析
通过对比原型文件和当前实现，发现段落样式存在以下差异：

1. **普通状态背景色**：当前使用 `SurfaceContainer`，原型要求使用 `BackgroundColor`
2. **其他样式**：宽度、圆角、内部编号形状和状态形状基本正确

### 解决方案
修改 `ContentLibraryScreen.kt` 文件中的段落样式实现：

1. **修改普通状态背景色**：将段落的普通状态背景色从 `SurfaceContainer` 改为 `BackgroundColor`
2. **验证其他样式**：确保宽度、圆角、内部编号形状和状态形状与原型一致

### 具体修改
在 `ContentCard` 函数中，修改段落的 `background` 颜色设置：

```kotlin
// 修改前
.background(
    when {
        paragraph.isCompleted -> Color(240, 253, 244)
        paragraph.isCurrent -> Color(255, 248, 250)
        else -> SurfaceContainer
    }
)

// 修改后
.background(
    when {
        paragraph.isCompleted -> Color(240, 253, 244)
        paragraph.isCurrent -> Color(255, 248, 250)
        else -> BackgroundColor
    }
)
```

这样可以确保段落的样式与原型完全一致，包括颜色、宽度、圆角、内部编号的形状和状态的形状。
