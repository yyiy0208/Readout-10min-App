# 1.3 云端数据库配置计划

## 1. 任务概述
完成Supabase云数据库的配置，包括数据库表结构创建、存储桶配置和环境变量设置，为web和移动端应用提供数据支持。

## 2. 所需资源
- Supabase账号（用户需要自行注册）
- 项目凭证（Project URL和Anon Public Key）

## 3. 实施步骤

### 3.1 创建数据库架构文件
- **任务**：创建 `SUPABASE_DATABASE_SETUP.sql` 文件，定义所需的4个核心表
- **内容**：
  - `content` 表：存储文章基本信息
  - `paragraphs` 表：存储拆分后的段落
  - `progress` 表：存储用户阅读进度
  - `practice_records` 表：存储练习记录
  - 表间关系和索引配置
  - RLS（Row Level Security）策略设置

### 3.2 更新开发计划状态
- **任务**：将任务1.3的状态从`pending`更新为`in_progress`

### 3.3 创建环境变量配置文件
- **任务**：为web和移动端创建环境变量模板文件
- **内容**：
  - `web/.env.example`：网页端环境变量模板
  - `android/local.properties.example`：移动端环境变量模板

### 3.4 提供用户操作指南
- **任务**：详细说明用户需要执行的Supabase配置步骤
- **内容**：
  1. 注册/登录Supabase账号
  2. 创建新项目
  3. 运行SQL脚本创建表结构
  4. 创建存储桶
  5. 获取项目凭证
  6. 配置环境变量

## 4. 预期结果
- 成功创建Supabase项目和数据库表
- 配置好存储桶用于文件上传
- 应用能够连接到Supabase数据库
- 完成任务1.3的开发计划
